# Amaso Companion (macOS menu bar)

**Not a dashboard.** The dashboard is the PWA at `dashboard.amaso.nl`.
This is a tiny helper that lives in the macOS menu bar and exists only to
run things a browser can't — shell commands, filesystem access, system
audio ducking — on commands dispatched from the cloud.

## Posture

- **No dock icon, no app menu, no main window.** `LSUIElement=true` in
  `Info.plist` (set by `electron-builder` via `mac.extendInfo`) and
  `app.dock.hide()` at runtime.
- A single `Tray` in the menu bar is the entire UI surface.
- Clicking the tray toggles a small frameless popover anchored to the
  icon. First run → login form. Signed in → connection status + sign out.
- Right-click (or long-press) the tray for the text menu: open dashboard,
  sign in/out, check for updates, quit.
- Single-instance lock — launching twice just reveals the existing tray.

## Files

| File | Purpose |
| --- | --- |
| `main.js` | Tray, popover window, IPC handlers, lifecycle, auto-update, session persistence (via `safeStorage`), VAD-driven ducking. |
| `preload.js` | Exposes `window.companion.{login, status, signOut}` to `login.html`. |
| `login.html` | Minimal vanilla-JS popover UI — login form, then status panel. |
| `audio-duck.js` | AppleScript-based system volume ducking, invoked from `dispatchCommand()` and from the local VAD. |
| `vad-preload.js` | Bridges the hidden VAD renderer to main via IPC (`amaso:vad:state`). |
| `vad.html` | Hidden window that runs a Web Audio RMS voice-activity detector on the default mic. |
| `assets/trayIcon*Template.png` | Menu-bar icons for the three states (idle/disconnected, connected, thinking). Generated by `node scripts/generate-tray-icons.mjs` — checked into the repo; regenerate whenever you tweak the glyph. macOS auto-inverts any file whose name ends in `Template` for light/dark menu bars. |

## Mic-aware ducking

On launch, a hidden `BrowserWindow` loads `vad.html`, grabs the default
microphone via `getUserMedia`, and runs a 20 Hz RMS detector. When the RMS
crosses the threshold for long enough, it posts `{ speaking: true }` to
main, which calls `duckOthers(0.25)`. Stop talking → after a short hang
window, `{ speaking: false }` → `restoreOthers()`.

This runs locally and requires no dashboard connection. macOS prompts for
microphone access the first time; the tray menu item **Mute others while
I'm talking** toggles the feature (persisted to `settings.json` in the
user-data folder). If the mic permission is denied the tray menu shows a
"Mic access denied" hint with a pointer to System Settings → Privacy.

The same `audio.duck` / `audio.restore` commands can still arrive via the
cloud channel once `connectToCloud()` is implemented — both paths share
`audio-duck.js`, which refcounts so rapid toggles can't leave the system
permanently turned down.

## Command channel

`connectToCloud()` in `main.js` opens a long-lived WebSocket to
`${AMASO_URL}/api/companion`, authenticating via the stored session
cookie, and pumps incoming messages into `dispatchCommand`. Protocol +
reconnect policy are documented inline in `main.js` and mirror the server
in `lib/companion-ws.ts`. Current handlers:

- `{ type: "audio.duck", level }` — lower other apps' volume.
- `{ type: "audio.restore" }` — put it back.
- `{ type: "shell.exec", cmd, cwd? }` — run a command in the user's login
  shell. Returns `{ stdout, stderr, exitCode, signal, timedOut, … }`.
  Capped at 30 s wall time and 1 MB per stream; output beyond that is
  truncated with a `*Truncated` flag set.
- `{ type: "fs.read", path }` — read a local file as the signed-in user.
  Returns `{ content, size, bytesRead, truncated, path }`. `~` is
  expanded. Capped at 10 MB; larger files are truncated.

Future (non-exhaustive):

- `{ type: "app.open", bundleId }` — open a native macOS app.

## Dev

```bash
cd electron
npm install
npm start                                    # talks to dashboard.amaso.nl
AMASO_URL=http://localhost:3737 npm start    # point at a local dev server
```

The three tray icons live in `assets/` and are regenerated from code by
`node scripts/generate-tray-icons.mjs` (run from the repo root). If you
ever wipe the directory, CI does the same thing automatically before the
build step.

## Release

```bash
cd electron
npm run dist        # arm64 + x64 DMGs into ./dist, no upload
npm run publish     # requires GH_TOKEN; publishes to GitHub Releases
```

CI (`.github/workflows/build-companion.yml`) builds both arches on every
push to `main`; trigger the workflow manually with `release: true` to
attach the DMGs to a fresh GitHub Release. Filenames are pinned to
`amaso-companion-arm64.dmg` / `amaso-companion-x64.dmg` (see
`mac.artifactName` in `electron/package.json`), so the dashboard's
"Install companion" buttons can resolve them via the stable
`releases/latest/download/<asset>` redirect — no manual URL wiring on
each version bump.
